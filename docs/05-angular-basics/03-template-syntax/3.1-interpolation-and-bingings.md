# Interpolation Vs Binding

String interpolation uses double curly braces `{{ }}` for one-way binding from component to view, while property and event bindings enable two-way and interactive data flow.

## String Interpolation Basics

String interpolation embeds component properties, expressions, or method calls directly into HTML text. Angular evaluates the expression inside `{{ }}`, converts it to a string, and inserts it safely (escaping HTML). For example, `{{ title }}` displays the `title` property value.

Use it for simple outputs like `{{ user.name + ' (' + user.age + ')' }}` or method calls like `{{ getFullName() }}`.

## Property Binding

Property binding uses `[property]="expression"` to set DOM element properties dynamically from the component. Unlike interpolation, it supports non-string values like booleans or objects, such as `[src]="imageUrl"` or `[disabled]="isFormInvalid()" [].

It flows data one-way from component to DOM, updating automatically on changes.

## Event Binding

Event binding captures DOM events with `(event)="handler()"` syntax, like `(click)="onSave()"` or `(input)="updateValue($event)"`. The `$event` object passes event details to the handler method.

Combine with property binding for responsive UIs, such as `[value]="searchTerm"` and `(keyup)="filterData($event)"`.

## Hands-On Example

Create `app.component.ts`:

```typescript
import { Component } from "@angular/core";

@Component({
  selector: "app-root",
  template: `
    <h1>{{ title }}</h1>
    <p [innerText]="message">Fallback text</p>
    <img [src]="imageUrl" [alt]="imageAlt" (click)="toggleImage()" />
    <button (click)="updateMessage()">Update</button>
  `,
})
export class AppComponent {
  title = "Bindings Tutorial";
  message = "Hello via property binding!";
  imageUrl = "https://via.placeholder.com/300";
  imageAlt = "Sample Image";
  showImage = true;

  toggleImage() {
    this.showImage = !this.showImage;
    this.imageUrl = this.showImage ? "https://via.placeholder.com/300" : "";
  }

  updateMessage() {
    this.message = `Updated at ${new Date()}`;
  }
}
```

Run `ng serve` to see interpolation render text, property binding set attributes, and event binding trigger actions.

## Best Practices

- Avoid complex logic in `{{ }}`, use methods or pipes instead.
- Prefer property binding over `[attr.property]` for native properties.
- Sanitize inputs to prevent XSS, as Angular auto-escapes interpolation.

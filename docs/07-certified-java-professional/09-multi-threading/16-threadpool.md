# Threadpool

Threadpool is a collection of pre-initialized threads.

## Why Threadpool
1. Optimized resource management: Creation and destructio of thread takes time.
2. Speed up reponse time
3. Control over thread count

## Executor Framework
- was introduced in Java 5 as part of java.util.concurrent.
- it simplifies development of concurrent applications by abstracting away the complexities involved in creating and managing threads.
- it provide 3 interfaces
    - 1. Executor
    - 2. ExecutorService
    - 3. ScheduledExecutorService


## Why Executor Framework?
1. Manual Thread Management: developers had to manually start the threads.
2. Resource Management:
3. Scalability
4. Thread Reuse
5. Error Handling
